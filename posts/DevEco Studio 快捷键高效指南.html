DevEco Studio 快捷键高效指南（按使用频率排序）

前言

DevEco Studio 是 HarmonyOS 应用开发的官方 IDE，它基于 IntelliJ IDEA 平台构建，因此快捷键体系与 IntelliJ IDEA 高度兼容。熟练掌握快捷键是提升编码效率、减少鼠标操作、专注于逻辑实现的关键一步。

本文档按照日常开发中的使用频率，将快捷键分为编辑、查找替换、导航、重构、运行调试等核心模块，并提供了实用技巧，帮助新手快速上手，老手查漏补缺。

提示：下方表格中 ⌘ 代表 Command 键，Option 代表 Alt 键，Control 代表 Ctrl 键。

七、自定义快捷键指南

若默认快捷键与你的系统全局热键冲突，或你更习惯其他编辑器（如 VS Code）的键位布局，可按以下步骤调整：

  1. 打开设置面板：Ctrl + Alt + S (Windows/Linux) 或 ⌘ + , (macOS)。
  2. 左侧导航栏选择 Keymap。
  3. 在搜索框中输入功能名称（例如 Reformat Code）。
  4. 右键点击对应的搜索结果，选择 Add Keyboard Shortcut。
  5. 在弹窗中按下你希望使用的新组合键，点击 OK 保存。

八、实战工作流建议

除了死记硬背快捷键，更推荐将它们融入开发流程的肌肉记忆中：

1. 代码修复与清理流

  1. 将光标放在报红或有警告的位置。
  2. 按下 Alt + Enter 查看 IDE 提供的修复方案（如导入包、添加缺失方法）。
  3. 完成功能后，在文件内按下 Ctrl + Alt + O 清理多余 Import。
  4. 最后按下 Ctrl + Alt + L 统一格式化代码缩进。

2. 逻辑调试定位流

  1. 运行项目：Shift + F10。
  2. 遇到预期外的表现，切换到调试模式：Shift + F9。
  3. 在行号右侧单击打上断点。
  4. 触发该逻辑后，使用 F8 逐行跳过，F7 深入方法内部查看变量值变化。

3. 模板代码生成流

  1. 在 .ets 文件中新建一个 Class。
  2. 在 Class 大括号内按下 Alt + Insert (Win) 或 ⌘ + N (Mac)。
  3. 选择 Constructor、Getter and Setter 或 toString，一键生成样板代码，大幅减少手写工作量。

结语：这份指南覆盖了日常鸿蒙应用开发中 95% 以上的高频操作。建议初学者先重点掌握 第一部分（编辑） 和 第二部分（查找），随着项目经验的增加，逐步练习重构与调试快捷键。坚持使用一周，你会发现敲代码的流畅度将有质的飞跃。
